About👀

Dear Diary has many built-in features that handle user-submitted data, to make it safe for storing your personal thoughts.These features escape possible dangerous characters in requests, making the data safe for the application to use.

Password Management🔑

Dear Diary endeavors to provide a secure and flexible set of tools for managing user passwords. Your passwords are stored using a PBKDF2 algorithm with a SHA256 hash; this is a password stretching mechanism recommended by NIST (National Institute of Standards and Technology).

Attacks❌

Dear Diary uses the CSRF middleware which provides easy-to-use protection against Cross Site Request Forgeries. This type of attack occurs when a malicious website contains a link, a form button or some JavaScript that is intended to perform some action on out website, using the credentials of a logged-in user who visits the malicious site in their browser. CSRF ensures that GET requests (and other ‘safe’ methods) are side effect free. Requests via ‘unsafe’ methods, such as POST, PUT, and DELETE, are rejected if they do not contain the specific CSRF tokens that inbuilt into our site.

Dear Diary also provides other fomrs of protection such as:

  • - SQL injection protection
  • - Clickjacking protection
  • - Enforcing SSL/HTTPS encryption
  • - Host header validation
